tls握手流程对比

发表于 2020-11-11 更新于 2021-06-20

几个TLS协议的握手流程备忘

1. 标准流程对比

1.1. TLS1.0,1.1,1.2,GMv1.1

1.1.1. 完整握手

1.1.2. 会话恢复

1.2. TLS1.3

1.2.1. 完整握手

1.2.2. DHE共享密钥不合适时候的握手

1.2.3. 会话恢复和PSK

1.2.4. 0-RTT

1.3. DTLS1.2

1.3.1. 防DDOS攻击

1.3.2. 完整握手flight

1.3.3. 会话恢复flight

2. 简化

握手过程主要目标是2个：

  1. 协商各种加密参数
  2. 认证，包括client和server认证

2.1. TLS1.0,1.1,1.2,GM1.1

2.1.1. 完整握手

2.1.2. 会话恢复

2.2. TLS1.3

2.2.1. 完整握手

2.2.2 会话恢复

2.2.3. 0-RTT

3. 总结

TLS1.0,1.1,1.2,GMv1.1, DTLS1.2完整握手基本一样，都是先发送证书，再交换密钥，最后进行握手认证(Finished)；会话恢复也都一样，先发送 session id 或 ticket 恢复密钥和认证，在进行握手认证(Finished)。
TLS1.3的先交换密钥，再发送证书，最后进行握手认证(Finished)。基本差不多。
